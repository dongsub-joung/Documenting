# 참조

- 목적 :

  1. 프로젝트 독립적인 라이브러리들을 사용하고 싶을 때

  2. a. 의 경우에서, 라이브러리 소스파일에 접근하고, 수정할 필요가 있을 때

- 효과

  - 라이브러리 코드에 "미리 컴파일된 헤더"를 추가할 필요 없음
  - 독립적인 라이브러리가 되었기 때문에 다른 프로젝트에서 사용 가능
  - 모듈화 되었으므로 단위 테스트 가능
  - 라이브러리를 수정하지 않는다면 라이브러리 크기만큼의 컴파일 시간 단축
